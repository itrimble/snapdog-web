What the app actually does.

The site now describes the actual product surface instead of generic landing-page copy.

Capture modes

Fullscreen, window, region, delayed capture, display-under-cursor, stitch capture, and auto-scroll capture are represented in the app source.

Precision pickers

Region picker includes magnifier loupe, live pixel coordinates, size badge, Esc cancellation, and window hit-testing by z-order.

Editor pipeline

Canvas annotation, blur, resize, border, shadow, watermark, metadata stripping, and hashing live in the image/effects pipeline.

Text and code tools

Vision-powered OCR copies recognized text to the pasteboard; barcode detection supports QR, Aztec, Data Matrix, and PDF417.

Recording stack

AVAssetWriter-backed video/GIF recording, HUD state, system audio, and webcam preview managers are present in the codebase.

Workflow automation

WorkflowStore, DestinationStore, webhook settings, storage settings, hotkey settings, history, notifications, and keychain-backed secrets are wired as first-class modules.

The workflow.

A user-facing flow distilled from the current app architecture.

Choose a capture mode from the menu bar or hotkey.
Use SnapDog’s picker overlays for exact region/window targeting.
Annotate or transform the image in the editor.
Save locally, record history, run workflow actions, or prepare destinations/webhooks.

Product foundation.

What supports the product experience behind this release.

  • SnapDog’s product scope includes ScreenCaptureKit capture, OCR, QR generation/decoding, GIF/video export, history, and settings.
  • The implementation includes dedicated capture, recording, workflow, destination, watch-folder, permissions, and settings systems.
  • The product direction includes App Sandbox, Hardened Runtime, and macOS 14+ App Store-oriented development.

Current status

Development preview. Pricing/distribution can be finalized when the App Store package is ready; current messaging intentionally avoids unsupported ratings or download claims.
